# AGENTS.md

## Codegen (required before build)

Two codegen steps must run whenever their source files change. Generated files are committed — **do not edit them by hand**.

```bash
templ generate   # .templ → components/*_templ.go
sqlc generate    # schema.sql + query.sql → db/{db,models,query.sql}.go
go build ./...
```

Shortcut for full regenerate + verify:

```bash
templ generate && sqlc generate && go build ./...
```

`templ` is declared as a Go tool dependency (`go.mod` `tool` directive). If the standalone binary is missing, use:

```bash
go tool templ generate
```

## Module path

`example.com/ical` — use this for all internal imports (not a real domain, just the module name in `go.mod`).

## Package layout

Flat single module. Key packages:

| Path | Role |
|---|---|
| `main.go` | Entrypoint: opens SQLite, embeds `schema.sql`, registers routes, starts HTTP |
| `db/` | **Generated by sqlc.** DBTX interface, `Queries` struct, `Event` model |
| `components/` | **Generated by templ.** Server-side HTML components (layout, forms, event list/row) |
| `handlers/` | HTTP route registration and all request handlers |
| `render/` | RFC 5545 iCal (`.ics`) rendering from `[]db.Event` |
| `timetype/` | Custom time type: JSON/SQL/iCal serialization, UTC normalization; only package with tests |

## Architecture note

The `.github/copilot-instructions.md` file describes an **old** architecture (JSON API, `event/event.go`, `static/index.html`). Ignore it. The current stack is:

- **UI**: server-rendered HTML via templ + HTMX (no JS build pipeline; Tailwind and htmx loaded from CDN)
- **DB**: sqlc-generated type-safe queries against SQLite (`modernc.org/sqlite`, pure Go, no CGo)
- **Routes**: form-based HTMX partials at `/events`, `/events/{uuid}/edit`, etc. — not a JSON API

## Key conventions

- **iCal output must use CRLF (`\r\n`)** per RFC 5545. `render.RenderICS` already does this; maintain it in any additions.
- **`timetype.CustomTime.ICS()`** always outputs UTC (`20060102T150405Z`). The `timezone` column in SQLite is stored but not currently used in iCal output.
- **Location field is nullable**: `sql.NullString` — empty form input → NULL in DB, not empty string.
- **Schema is embedded and applied at startup** via `//go:embed schema.sql` + `CREATE TABLE IF NOT EXISTS`. There is no migration system.
- **SQLite file**: `./data/ical.db` relative to the working directory (gitignored except for `.gitkeep`).
- **Event PKs**: UUID strings generated by the caller using `github.com/google/uuid`.

## Testing conventions

- Use `_test.go` in the same package (not `package foo_test`).
- Use `t.Helper()` for shared assertion helpers.
- DB tests: open `:memory:` with `sql.Open("sqlite", ":memory:")` and close with `t.Cleanup()`.
- Handler tests: use `httptest.NewRecorder` or `httptest.NewServer`.

## Known inconsistency

`query.sql` has 9 parameters for the INSERT (including `timezone` and `all_day`), but `db/query.sql.go` has 7 (those columns omitted). The generated file is the runtime source of truth. `schema.sql` defines `timezone` and `all_day` columns that are never populated by current handler code.

## Toolchain quirk

`go.mod` declares `go 1.25.1` (prerelease). This matches the local toolchain but may fail in environments pinned to a stable Go release.

## No CI, no task runner

No `.github/workflows/`, no `Makefile`, no `Taskfile`, no pre-commit config.
